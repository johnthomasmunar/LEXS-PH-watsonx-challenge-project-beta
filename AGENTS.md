## ⚡ New Project Setup (5 Steps)

> ⚠️ **Always use `npm`** — never `yarn`. Having both `yarn.lock` and `package-lock.json` in the same project breaks Next.js 16 Turbopack.

### Step 1 — Scaffold (terminal)

```bash
npx create-next-app@latest my-course --typescript --tailwind --app --src-dir=false
cd my-course
mkdir -p scripts
```

### Step 2 — Manual file copy (from this project)

```
AGENTS.md              →  my-course/AGENTS.md
scripts/bootstrap.mjs  →  my-course/scripts/bootstrap.mjs
```

### Step 3 — Run bootstrap (terminal)

```bash
node scripts/bootstrap.mjs
```

`node scripts/bootstrap.mjs` automatically:
1. **Installs all npm dependencies** — `@carbon/react@1.33.0`, `sass`, `mammoth`
2. **Creates all folders** — `data/`, `scripts/`, `types/`, `components/`, `app/course/`, `public/`
3. **Writes all source files** — every component, page, type, and parse script
4. **Patches `package.json`** — adds `parse-course` and `bootstrap` scripts
5. **Creates placeholder `data/course-data.json`** — so the app compiles before you have a `.docx`

### Step 4 — Drop your content files in (manual)

```
data/course_template.docx   ← your Word document (required)
public/banner.jpg            ← module banner background (required)
public/placeholder.jpg       ← OPTIONAL: fallback for broken docx image URLs
```

### Step 5 — Run

```bash
npm run dev
```

Open **http://localhost:3000** 🎉

> `npm run dev` automatically parses the `.docx` and starts the dev server in one command.

> **Updating content later?** Edit your `.docx`, drop it back in, then run `npm run dev` again.

> **No AI needed** — all 5 steps are just terminal commands and file copies. Bob is optional and only helps with questions, code changes, and debugging.

## Goal

Convert a Word document (`.docx`) into a fully functional learning course web app using:
- **Next.js 16 App Router** (static, no backend)
- **@carbon/react 1.33.0** for UI shell (Header, SideNav)
- **Plain React** for interactive components (Tabs, Accordion) — Carbon versions are broken in React 19
- **Tailwind CSS v4** for utility layout classes
- **mammoth** to parse the `.docx` at build time into `data/course-data.json`

The `.docx` is the single source of truth. Sections, modules, component hints (Accordion/Tabs), and image URLs all come from the document. Images from the document (e.g. IBM internal URLs) are replaced with a local `public/placeholder.jpg` at render time.

## Word Document Format

The parser (`scripts/parse-docx.mjs`) expects this structure:

```
COURSE INFORMATION
Course Title:  <title>
Course Code:   <code>
Duration:      <duration>
Delivery Mode: <mode>
Course Owner:  <email>

COURSE DESCRIPTION
<paragraph text>

Course Navigation
Type: Side bar
Module  Topic                            Duration
1       Introduction to Web Development  90 mins
2       HTML5 Fundamentals               120 mins
...
Total Duration: ...

Module 1: Introduction to Web Development
Duration: 4 Hours

Introduction
<body text>
Image: https://...
Suggested Layout:
Left side: Text
Right side: Image

Learning Objectives
<body text>
Image: https://...
Suggested Layout:
Left side: Image
Right side: Text

Types of Web Development
<body text>
Component: Accordion
Common technologies include:
HTML
CSS
JavaScript
Responsibilities include:
Designing page layouts
...

Module 2: HTML5 Fundamentals
...
Component: Tabs
Tab 1 – DOCTYPE
Purpose of the <!DOCTYPE html> declaration
HTML5 document standard
Tab 2 – Head Section
Page title
...
Body Section
Visible page content
...
```

### Parser rules
- `COURSE INFORMATION` → `COURSE DESCRIPTION` block → course metadata fields
- `Course Navigation` table → module list (number, title, duration)
- `Module N: Title` headers → module content blocks
- Known section headings: `Introduction`, `Learning Objectives`, `Types of Web Development`, `HTML Document Structure` — extend `KNOWN_SECTION_HEADINGS` set in the script for new headings
- `Component: Accordion` → next short lines become accordion items; lines ending `:` are group titles
- `Component: Tabs` → lines matching `/^Tab \d+/i` or short title-cased lines (≤35 chars) become tab headers
- `Image: https://...` lines → URL captured into `section.image` (type `string | false`); rendered with `onError` fallback to `public/placeholder.jpg`
- All noise lines filtered: `Suggested Layout`, `Left side:`, `Right side:`, `https://...` etc. (`Image:` lines are NOT filtered — they are captured before the noise filter runs)

## TypeScript Types (`types/course.ts`)

```ts
export interface Section {
  heading: string;
  body: string;
  image: string | false;  // URL string from docx, or false if no image
  layout: "text-left image-right" | "image-left text-right" | "text-accordion" | "text-tabs" | null;
  component: "Accordion" | "Tabs" | null;
  items: string[];
}

export interface Module {
  id: string;
  number: number;
  title: string;
  duration: string;
  sections: Section[];
}

export interface Course {
  title: string;
  code: string;
  duration: string;
  deliveryMode: string;
  owner: string;
  description: string;
  modules: Module[];
}
```

## Carbon Import Rules (CRITICAL)

### Rule 1 — Always `"use client"`
Every file that imports from `@carbon/react` must have `"use client"` as its first line.
Carbon uses React context internally and will crash in Server Components.

### Rule 2 — Import UIShell components via `require()` from ES path
`@carbon/react` 1.33.0 barrel exports are incomplete for React 19. UIShell components
(`Header`, `HeaderName`, `SideNav`, `SideNavItems`, `SideNavLink`) and `Tag` either have
no `.d.ts` or fail TypeScript with missing `placeholder`/`onPointerEnterCapture` props.

**Always use this pattern:**
```ts
/* eslint-disable @typescript-eslint/no-require-imports */
const Header = require("@carbon/react/es/components/UIShell/Header").default as React.ComponentType<{
  "aria-label": string; className?: string; children: React.ReactNode;
}>;
const HeaderName = require("@carbon/react/es/components/UIShell/HeaderName").default as React.ComponentType<{
  href?: string; prefix?: string; children: React.ReactNode; className?: string;
}>;
const SideNav = require("@carbon/react/es/components/UIShell/SideNav").default as React.ComponentType<{
  isFixedNav?: boolean; expanded?: boolean; isChildOfHeader?: boolean;
  "aria-label"?: string; className?: string; children: React.ReactNode;
}>;
const SideNavItems = require("@carbon/react/es/components/UIShell/SideNavItems").default as React.ComponentType<{
  children: React.ReactNode;
}>;
const SideNavLink = require("@carbon/react/es/components/UIShell/SideNavLink").default as React.ComponentType<{
  isActive?: boolean; onClick?: () => void; href?: string; children: React.ReactNode;
}>;
/* eslint-enable @typescript-eslint/no-require-imports */
```

### Rule 3 — Import Carbon CSS once per client component
```ts
import "@carbon/styles/css/styles.css";
```
Do NOT import `@carbon/react/index.scss` inside a `.css` file — it requires `.scss`.

### Rule 4 — Never use Carbon `Tabs`, `TabList`, `TabPanels`, `TabPanel`
**Symptom:** `TypeError: Cannot read properties of undefined (reading 'disabled')`

**Root cause:** Carbon's `Tabs` reads `.disabled` from `tabs.current[selectedIndex]`.
React 19's concurrent renderer populates the ref array asynchronously — it is `undefined`
at first paint, crashing the component.

**Fix:** Use the plain React `TabsSection` component (see `components/ModuleContent.tsx`).
All styling uses inline `style` props so Carbon CSS cannot override them.

### Rule 5 — Never use Carbon `Accordion`, `AccordionItem`
Same class of crash — Carbon `Accordion` uses `AccordionContext` with internal state that
breaks under React 19's concurrent renderer.

**Fix:** Use the plain React `AccordionSection` component (see `components/ModuleContent.tsx`).

### Rule 6 — Use inline `style` props for all padding/spacing inside content components
Carbon's `@carbon/styles/css/styles.css` resets and overrides Tailwind utility classes.
**Any padding, margin, or spacing that must survive must use `style={{ ... }}`.**

Tailwind classes ARE safe for: layout (`flex`, `grid`, `overflow`, `w-*`, `h-*`),
colors on elements Carbon doesn't touch, and structural classes on wrappers Carbon ignores.

## Known Issues & Fixes

### SectionBlock keys must include module ID
`SectionBlock` must be keyed `${module.id}-${idx}` (not just `idx`) so React fully
remounts sections when switching modules. Without this, Carbon component internal state
leaks between modules.

### TabsSection must be keyed by heading
`<TabsSection key={section.heading} ...>` forces remount when tab content changes.

### No inline IIFEs in JSX
Never use `{(() => { return <Component/>; })()}` inside JSX render. React cannot reconcile
IIFE return values as children — crashes on re-render. Always extract to named components.

### globals.css must not import Carbon SCSS
`app/globals.css` only contains `@import "tailwindcss"`. Carbon styles are imported
per-component via `import "@carbon/styles/css/styles.css"`.

## Adding New Module Section Types

1. Add the new heading string to `KNOWN_SECTION_HEADINGS` in `scripts/parse-docx.mjs`
2. Add a `deriveLayout()` case if the section has a specific layout
3. Add rendering logic in the `SectionBlock` function in `components/ModuleContent.tsx`
4. Re-run `npm run parse-course`
